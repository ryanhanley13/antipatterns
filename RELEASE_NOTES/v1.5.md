The skill now ships ready to use in whatever tool you write in, not just Claude. A new one-page guide plus ready-to-paste files for ChatGPT and Gemini, alongside the packaged `.skill` for Claude and Manus.

## What's new

- **Use it in any tool.** A new `USING.md` walks through setup for Claude (import the `.skill`), Manus (import this repo's GitHub URL), ChatGPT (a **Project**: paste the condensed instructions and upload the full catalog), and Gemini (a **Gem**). Its headline pattern count is pulled live from the catalog, so it stays honest as the catalog grows.
- **Ready-to-paste files for ChatGPT / Gemini.** This release attaches two files alongside the `.skill`:

  - `chatgpt-instructions.md`: the condensed (under-8,000-character) zero-tolerance core, for a ChatGPT Project's custom-instructions field.
  - `full-instructions.md`: the full procedure + catalog, for a Gemini Gem or a ChatGPT Project's uploaded file.

  Both are in the assets below.

No catalog or detection changes since v1.4, this is a sharing release. (The `--discover` candidate-discovery mode shipped in v1.4.)

## For contributors

- `tools/export.py` now generates the committed `USING.md` guide as well as the `dist/` wrappers, all from `SKILL.md` + `ANTIPATTERNS.md`. A new `--check` mode plus a CI job keep `USING.md` in sync with the catalog.
- `release.yml` builds and publishes the community wrappers to the Release on every `v*` tag. The repo URL is detected from `git`, so a fork's generated guide points at the fork, not upstream.
- Test suite 86 to 95.
